# CLI 术语表（任务导向）

本术语表用于统一 CLI 默认输出与 `--verbose` 诊断输出，避免同概念多词混用。

## 基本原则

- 用户可见文案优先使用“结果词 + 动作词”。
- 同一概念只用一个主词，必要时在括号给出兼容词。
- 新增文案必须复用本表术语，不得自行发明同义词。

## 核心术语（中英映射）

| English | 中文主词 | 备注 |
| --- | --- | --- |
| watermark | 水印 | 不用“标记”“印记”替代 |
| detect | 检测 | 动词统一为“检测” |
| embed | 嵌入 | 动词统一为“嵌入” |
| key slot | 密钥槽位 | 不简写为“槽” |
| evidence | 证据记录 | 列表可简称“证据” |
| mapping | 映射 | 不用“绑定”替代 |
| fallback | 回退 | 诊断中可用“回退路径” |
| verify | 校验 | 对应 verify/verification |
| invalid | 无效 | 用于校验失败或非法结果 |
| unavailable | 不可用 | 用于能力不可用，不用于失败 |

## 文案模板

- 默认用户输出：
  - 第一句：发生了什么（结果）。
  - 第二句：下一步动作（可执行命令或检查项）。
  - 第三句：仅在必要时补充原因。
- `--verbose` 诊断输出：
  - 保留完整技术信息。
  - 术语必须与本表一致。

## 禁止项

- 默认文案直接暴露内部字段名（例如 `slot_hint`、`scan_count`）。
- 默认文案只有报错，没有下一步动作。
- 同一概念在同一语言内混用多个主词（例如“映射/绑定”并存）。
